What's in a URL?

I think one piece of the technology stack often overlooked is the URL. A clean and short URL can be so valuable, but what if there was more to it?
Take a look at this image here. I want you to ask yourself, are there any parts of this that I don't know? There's no test, just keep it in mind.
If I were to guess the parts of a URL that most people don't know the name of, it would be as follows:
  1. Fragment
  2. Query String Separator
  3. Scheme
  4. Query String
The most interesting one here is probably the Fragment. Lets learn a bit about it.
The Fragment specifies a location within a page. So you might go to the page, but this dictates where on that page you will open up to.
Name Attributes of HTML elements are the actual data points searched for with the fragment value. The easiest example is the element ID. ID is an HTMl element attribute that you can set, but you can only set an ID to one and only one element on a single page.
Now you can also use the fragment to select certain characters or lines which is pretty cool, but thats neither here nor there.
One of the most interesting things about fragments, is that all of the searching is done client-side. This means that your browser doesn't actually make a request to the web server for that specific element name. What the browser does instead is make a call for the full html file, but then once it receives the response back, it does the searching locally.
If you change the fragment, it doesn't result in a reload because its all local, but it is supposed to add an entry in your browser history showing that you visited the URL with that fragment.
One last cool thing:
If you are in Chrome or Edge(these are the only two browsers I have tested this in), select/highlight some text on the page, then right click and select "Copy Link to Highlight". This will use some wizardry to select text through an element content search instead of an attribute search and then actually modify the style of the text as well to show a highlight. Really cool feature for sending pages to somebody with specific information to show.
